## AGPL 对你意味着什么

### 你可以

- **阅读与学习**源代码
- **本地运行**软件（个人或公司内部使用）
- **修改**代码供自己使用
- 在遵守 AGPL 义务的前提下 **再分发**（含修改版）
- 通过 Pull Request **贡献**改进（推荐）

### 在分发或提供网络服务时，你必须

若你 **分发** 本软件，或基于本代码 **对外提供网络服务**（例如托管版 SaaS）：

1. **提供源代码** — 包括你的修改，远程使用服务的用户有权获得对应源码
2. **保持相同许可** — 衍生作品需继续在 AGPL 下发布（或在适用情况下使用兼容许可）
3. **保留声明** — 版权与许可声明不得删除
4. **说明修改** — 若修改了文件，应注明修改内容与日期

AGPL 在 GPL 基础上将 copyleft 延伸到 **网络使用**：远程用户访问修改版服务时，有权获得相应源代码。
