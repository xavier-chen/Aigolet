## What AGPL means for you

### You may

- **Read and study** the source code
- **Run** the software locally for personal or internal business use
- **Modify** the code for your own use
- **Distribute** copies (including modified versions), subject to AGPL obligations
- **Contribute** improvements back via pull requests (recommended)

### You must (when distributing or offering as a network service)

If you **distribute** this software or **offer it as a service** over a network (e.g. hosted SaaS built on this codebase):

1. **Provide source code** — including your modifications, to users who interact with the service remotely
2. **Keep the same license** — derivatives must remain under AGPL (or a compatible license where applicable)
3. **Preserve notices** — copyright and license notices must stay intact
4. **Document changes** — if you modify files, note what changed and when

The AGPL extends GPL’s copyleft to **network use**: users who access a modified version over the network are entitled to receive the corresponding source.
